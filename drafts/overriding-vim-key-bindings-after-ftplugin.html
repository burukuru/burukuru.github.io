Overriding Vim key bindings after ftplugin

Posted on Thu 22 June 2017 in Linux

I recently started using vim-terraform, it checks the syntax on save and keeps the config neat - great stuff! One problem I had was it remapped '' to 'za' which folds/unfolds functions. This was rather annoying as I use for bringing up the buffer list.

In order to unmap/remap this, I had to create ~/.vim/after/ftplugin/terraform.vim with this content:

nnoremap <buffer> <Space> :BufExplorerHor<CR>

This is because the plugin uses an after-directory which is only run after the file type (ftplugin) is detected. Creating your own after-directory will override this.
